# Contributing to IASTG

Thank you for your interest in contributing to the Infrastructure, Active Directory, and Linux Security Testing Guide.

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/new-control`)
3. **Follow the control template** - All controls must follow the 10-section template
4. **Test your changes** - Ensure technical accuracy
5. **Submit a pull request**

## Control Template

Each control must include these sections in order:

1. Summary
2. Risk
3. Preconditions
4. Test Objectives
5. Test Methodology
6. Expected Secure Configuration
7. Evidence to Collect
8. Impact
9. MITRE ATT&CK Mapping
10. Remediation Guidance

## Style Guidelines

- Use professional, normative language
- Write each control as standalone (no cross-references)
- Include specific, actionable testing steps
- Provide concrete remediation guidance
- Map to MITRE ATT&CK where applicable

## Questions?

Open an issue for discussion before starting major contributions.
